# Лабораторная работа: Ускорение вычислений множества Мандельброта с помощью SIMD-инструкций

## Цели и задачи
1. Реализовать программу с большим объёмом вычислений (отрисовка множества Мандельброта).
2. Исследовать влияние различных подходов к векторизации на производительность:
   - автоматическая векторизация компилятором на основе циклов с массивами (`Quad`);
   - ручная векторизация с использованием Intel Intrinsics (`AVX2`).
3. Оценить влияние флагов оптимизации компилятора (`-O0`, `-O2`, `-O3`).

## Параметры системы
| Параметр | Значение |
|----------|----------|
| Система | Windows 11 Pro |
| Процессор | AMD Ryzen 7 8845HS (Zen 4, 8 ядер / 16 потоков, базовая частота 3.3 ГГц, макс. буст 5.1 ГГц) |
| ОЗУ | 16 ГБ DDR5 |
| Компилятор | g++ (MinGW-w64) 15.1.0 |
| Опции компиляции | `-DBENCHMARK_MODE -march=native -O0` / `-O2` / `-O3` |
| Привязка к ядру | `start /affinity 4` (ядро CPU3) |
| Питание | От сети |

*Примечание:* Turbo Boost (Precision Boost) не был принудительно отключён, однако благодаря прогреву и усреднению по 400 кадрам результаты показали высокую повторяемость (отклонение <1% между запусками).

## Методика измерений и обработки данных

Для каждой реализации и каждого флага оптимизации было выполнено **10 последовательных замеров**. Внутри одного замера программа отрисовывала **400 кадров** (для AVX2 2000 кадров) в offscreen‑буфер, а измерялось **среднее число тактов процессора на один кадр**. Замеры проводились при питании от сети с привязкой к ядру CPU3.

Из полученных 10 значений отбрасывались минимальное и максимальное (для устранения случайных выбросов, вызванных, например, вмешательством ОС). Для оставшихся 8 значений вычислялись: Среднее арифметическое, Стандартное отклонение (несмещённая оценка, делитель `n−1 = 7`), которое принимается за погрешность измерения.

## Обсуждение результатов

### Влияние флагов оптимизации

- **Simple**: Переход с `-O0` на `-O2` дал ожидаемое ускорение в **2.8 раза** за счёт базовых оптимизаций. Флаг `-O3` не принёс дополнительного выигрыша.
- **Quad**: На `-O2` достигнуто колоссальное ускорение в **30 раз** относительно `-O0` — компилятор успешно автоматически векторизовал циклы с массивами. Однако флаг `-O3` привёл к **замедлению в 1.83 раза** по сравнению с `-O2`. с `-O3` получилась деоптимизация (причины будут исследованы далее). 
- **AVX2**: Ручная векторизация дала ускорение в **6.0 раз** на `-O2`. Флаг `-O3` практически не изменил результат.

### Анализ замедления `-O3` для Quad с помощью Godbolt

Для выяснения причин замедления Quad при `-O3` код вычислительного ядра был загружен в Compiler Explorer (godbolt.org) с флагами `-O2` и `-O3`. Сравнение ассемблерного листинга показало, что на `-O3` внутренний цикл был развёрнут в два раза, что увеличило давление на регистры. В результате появились инструкции сохранения векторов в стек (`vmovaps [rsp+...], ymm...`) и их последующей загрузки, отсутствовавшие в версии `-O2`. Дополнительные обращения к памяти нивелировали выигрыш от развёртки и привели к общему замедлению.

## Выводы

1. Автоматическая векторизация компилятором (`Quad`) способна превзойти по эффективности код с ручными AVX2‑интринсиками, если структура циклов позволяет компилятору «понять» намерения программиста.
2. Флаг `-O3` не всегда даёт прирост производительности; в ряде случаев он может приводить к **деоптимизации** (как в случае с `Quad`), поэтому выбор флагов должен подтверждаться измерениями.
3. Ручная векторизация (AVX2) обеспечивает стабильное и предсказуемое ускорение, однако требует тщательного анализа ассемблерного кода для достижения максимальной эффективности.
4. Методика измерений (прогрев, фиксация ядра, усреднение по 400 кадрам, исключение выбросов) позволила получить воспроизводимые результаты с относительной погрешностью от 0.02% до 0.5%.
